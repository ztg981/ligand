# Security Policy

## Supported Versions

Ligand is pre-1.1. Security fixes are applied to the current `master` branch and the latest published desktop/web build.

## Reporting a Vulnerability

Please report suspected vulnerabilities privately to the repository owner through GitHub's private vulnerability reporting feature if it is enabled. If private reporting is not enabled, open a minimal GitHub issue asking for a private contact path; do not include exploit details, secrets, tokens, private user data, or working attack payloads in a public issue.

Helpful details:

- Affected feature, platform, and build version.
- Impact and reproduction steps using test data only.
- Whether authentication is required.
- Any relevant request IDs or timestamps.

Please do not test against real user data, attempt destructive actions, access another person's account, run denial-of-service tests, or use automated scanners against production without written permission.

The project aims to acknowledge reports within 7 days, provide a remediation plan for confirmed issues, and coordinate disclosure after a fix is available.
